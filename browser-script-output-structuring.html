browser_script Output Structuring

The browser script can now emit separate channels for model data, transcript summaries, files, and images. The TUI no longer has to print the raw Python stdout block just to preserve data for the model.

model-visible: outputs TUI-visible: summary side-channel: images side-channel: artifacts fallback: bounded text

Key answer

image used to mean only “a tool.image event exists.” With the new metadata, screenshot captures render as screenshot: hn_front_page, explicit image attachments render as attached image: diagnostic, and old browser-script history falls back to browser image: label.

browser_summary

A top-of-script comment maps labeled structured outputs to compact transcript summaries.

emit_output

The helper emits full-fidelity data for the model. If the label has a summary spec, the runtime derives a transcript-safe record.

emit_image / screenshot

Image artifacts with source metadata. This tells the UI whether the image was captured from the browser or attached by script code.

End-to-End Flow

1

Python helper emits labeled data

The runtime parses # browser_summary: before execution. emit_output(..., label="..."), emit_image(...), and screenshot helpers append records to in-memory arrays and stream JSON events to Rust.

2

browser-use-browser drains deltas

BrowserScriptDelta collects text, outputs, summary, images, artifacts, and browser events while the background script is running.

3

browser-use-core records events

The core writes transcript-safe tool.output events, separate tool.image events, and model-visible structured parts. Transport lines like run_id are not needed in the visible transcript.

4

browser-use-tui renders deterministic labels

The transcript prefers summary. If no summary exists, it shows structured-output labels. Raw text is only a bounded fallback and runtime instruction lines are filtered.

Display Rules

Input Transcript row
summary.kind = "page" page: news.ycombinator.com - Hacker News
summary.kind = "clicked" clicked: Sign in -> gusto.com/login
summary.kind = "extracted" with message Sampled 5 comments from current thread
image.source = "screenshot" screenshot: hn_front_page
image.source = "emit_image" attached image: diagnostic
legacy browser-script image, no source browser image: latest_screenshot

Why “browser connected” Was Clean

That row came from a typed browser lifecycle event such as browser.reconnected, not from arbitrary tool stdout. The renderer maps that event type directly to one label: browser connected. There is no raw page-info dictionary to preview.

Why browser_script Was Messy

The browser script used to rely on printed Python values. The TUI only saw tool.output.text, so it previewed raw dictionaries, long URLs, run_id, and “still running” transport instructions. The new channels let the script keep data without making stdout the UI contract.

Model vs User Visibility

Channel Who needs it What it contains UI policy
outputs Model Full JSON data from the page or extraction task, emitted by emit_output Do not parse for primary display
summary User and transcript Compact action/result records derived from the top comment and linked by output_label Primary display surface parsed by the TUI
images Model and user artifacts Image path, MIME type, label, source Render source-aware labels
artifacts User and result handling Saved files, JSON outputs, screenshots as files Show counts or file pointers, not full bodies
text Fallback/debug stdout and legacy script output Bound, truncate, and filter runtime lines

Current Code Touchpoints

crates/browser-use-browser/src/lib.rs defines BrowserScriptOutput, parses # browser_summary:, streams linked output and derived summary events, and gives emit_image a source.

crates/browser-use-browser/src/browser_script_helpers.py marks screenshot captures with source: "screenshot".

crates/browser-use-core/src/lib.rs records transcript-safe events and preserves structured data for model-visible messages.

crates/browser-use-tui/src/transcript.rs turns structured records into deterministic rows and filters transport-only text.
