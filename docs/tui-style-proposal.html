Browser Use TUI style target — v2

runtime cockpit, not chat log

Flat surfaces, four colors, four primitives, six screens. Everything in this document maps 1:1 to something Ratatui can render — no shadows, no gradients, no rounded corners, no invented depth.

Changes from v1: depth tricks stripped, palette cut from 8 to 4, screens cut from 13 to 6, every primitive shown in 4 states, narrow/wide comparison for the cockpit screen, footers collapsed to /.

Color Tokens — 4 only

text#ECEEE8

default body text, active values, headings

muted#8A9088

labels, hints, footer text, separators

green#6CA876

ok, done, connected — desaturated on purpose, never focal

orange#FF6A1A

the only loud color — active work, progress, errors with [!]

No blue. No red. No yellow. Links get an underline, not a color. Errors are orange + [!]. Chrome colors (page, surface, panel, line, dim) exist but are structure, not vocabulary — agents and humans should never have to learn them.

Design Rules To Port

One loud color

Orange is the only saturated colour. Green and text are de-emphasised so orange always wins the eye. Don't argue with this.

Top owns state

Objective tree is fixed at the top and never scrolls. The first thing visible always answers "what is happening now."

Bottom owns control

Composer, chip row, metric band, progress bar are one grouped region. The eye knows where to act.

Same context, same place

Browser / model / session info appears in the metric band, once. Never repeat the same fact in a sticker, footer, and header.

Hairlines, not boxes

Grouping is a single dim grey line. +----+ is reserved for inputs only. Panels die.

Footer is just /

Setup keeps enter/esc because they're contextual. Every other screen uses / in the corner; the long key list lives inside the palette.

Progress is the heartbeat

The bar is always there. Idle = muted full, working = orange partial, done = green full, failed = broken orange pattern.

Tabular numerals, no ligatures

font-variant-numeric: tabular-nums and "calt" 0, "liga" 0. Numbers in the metric band must align across rows and screens.

Six screens, three states

Setup, Home, Running, Result, Failed, Palette. Running/Result/Failed are the same layout — failure isn't a new screen, it's a state.
